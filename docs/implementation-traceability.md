# NightOUT Implementation Traceability

This document maps the requirements from the NightOUT requirements and design documents to the current full-stack prototype.

Status legend:

- **Compliant**: implemented in the current prototype and usable in the demo.
- **Partially compliant**: some implementation exists, but important behavior is missing or simplified.
- **Not compliant**: no meaningful implementation exists yet.
- **Deferred by demo scope**: intentionally simplified for the first prototype.

2. Complete event filtering with date, venue category, price/entry condition, and a mobile filter panel with empty states.
3. Add ticket cancellation UI and friendly error handling for duplicate reservations and full/waiting-list flows.
4. Add a manager event creation/edit form connected to existing `POST /api/manager/events`, then add update/capacity endpoints.
5. Add a dedicated notifications page with read/unread state using the existing notification endpoints.
6. Add a pregame detail page matching the mockup and expose leave-room/joined-state behavior.
7. Add social summary support for friends attending events and display it on event detail cards.
8. Introduce lightweight service tests for ticket capacity, duplicate tickets, cancellation, and waiting-list promotion.
9. Document and implement a simple demo-level role guard for manager endpoints without adding complex authentication.
